# Go Microservice in Scratch Container

Минималистичный Go HTTP-сервис, упакованный в оптимизированный Docker scratch-образ (< 15MB).

## Функционал

- `/health` — возвращает `{"status": "ok"}`

## Конфигурация

Все настройки через переменные окружения:

| Переменная   | Описание              | По умолчанию     |
|-------------|-----------------------|------------------|
| `PORT`      | Порт HTTP сервера     | `8080`           |
| `LOG_LEVEL` | Уровень логирования   | `info`           |

## Сборка и запуск

### 1. Сборка Docker-образа

```bash
docker build -t go-microservice:latest .
```

Dockerfile автоматически:
- Запускает unit-тесты (`go test -v ./...`)
- Выполняет статическую компиляцию (`CGO_ENABLED=0`)
- Удаляет отладочную информацию (`-ldflags="-s -w"`)
- Создает не-root пользователя

### 2. Запуск с переменными окружения

```bash
docker run -e PORT=8080 -e LOG_LEVEL=info -p 8080:8080 go-microservice:latest
```

### 3. Проверка работы

```bash
curl http://localhost:8080/health
```

## Проверка размера образа

```bash
docker images go-microservice
```

Ожидаемый размер: **~5-8 MB** (значительно меньше 15 MB лимита).

## Локальное тестирование (без Docker)

```bash
go test -v ./...
```

## Архитектура образа

- **Stage 1 (Builder)**: `golang:1.22-alpine` — тесты + компиляция
- **Stage 2 (Final)**: `scratch` — только бинарник + CA-сертификаты
- **Безопасность**: приложение запускается от не-root пользователя (`appuser`)
